# Contributing to AIC Observatory

Thank you for your interest in contributing to AIC Observatory.

We welcome contributions from developers, researchers, and analysts.

---

# Ways to Contribute

You can contribute in several ways:

### Engineering

* signal ingestion systems
* pattern detection algorithms
* simulation infrastructure
* data pipelines

### Research

* civilization-scale modeling
* systemic risk analysis
* AI technology forecasting
* cognitive and social dynamics

### Governance

* ethics frameworks
* transparency mechanisms
* privacy safeguards

### Documentation

* tutorials
* architecture explanations
* research summaries

---

# Contribution Workflow

1. Fork the repository
2. Create a feature branch
3. Implement your changes
4. Add documentation and tests
5. Submit a pull request

Example:

```
git checkout -b feature/weak-signal-detector
```

---

# Pull Request Guidelines

Pull requests should include:

* clear description
* explanation of design decisions
* documentation updates
* tests when applicable
